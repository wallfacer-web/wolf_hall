# 人物研究提问指南

## 目的

这页用于沉淀两套以后可重复使用的提问路径：

- `NotebookLM`：适合做方法综述、研究路径整理、来源扎根和学术框架总结
- `GraphRAG / query_ollama.py`：适合直接向《狼厅》三部曲索引后的文本提问，抽取人物、关系、风格和叙事判断

简化理解：

- 先用 `NotebookLM` 问“应该怎样研究”
- 再用 `GraphRAG` 问“文本里具体怎样写”

## 一、NotebookLM 怎么问

### 适用场景

- 总结数字人文如何研究人物
- 总结某种方法的研究对象、常用技术和代表价值
- 让来源材料按主题归纳
- 生成“方法论层”的论文草稿

### 基本命令

先选中 notebook：

```powershell
$env:NOTEBOOKLM_HOME = "$env:USERPROFILE\.notebooklm-clean-home"
notebooklm use 99341476-8713-40c0-ae12-70f59cfd24e8
```

直接提问：

```powershell
notebooklm ask "数字人文如何研究文学人物？请围绕人物社会网络分析、情感弧线分析、人格画像、叙事聚焦/视角分析四种方法，总结其研究对象、常用技术、代表价值，并说明这些方法如何应用于《狼厅》三部曲中的克伦威尔、安妮·博林等人物研究。"
```

保存成笔记：

```powershell
notebooklm ask "数字人文如何研究文学人物？请围绕人物社会网络分析、情感弧线分析、人格画像、叙事聚焦/视角分析四种方法，总结其研究对象、常用技术、代表价值，并说明这些方法如何应用于《狼厅》三部曲中的克伦威尔、安妮·博林等人物研究。" --save-as-note --note-title "数字人文：人物研究方法"
```

### NotebookLM 提问模板

#### 模板 1：方法综述

```text
请按方法分类总结[主题]，每类方法说明：
1. 研究对象
2. 常用技术
3. 代表价值
4. 对《狼厅》三部曲的适用场景
```

#### 模板 2：论文写作型

```text
请基于当前 notebook 的来源，写一段适合论文使用的分析，要求：
1. 先总结研究共识
2. 再指出可用于《狼厅》的方法
3. 明确区分“来源扎根结论”和“应用推演”
4. 用中文输出，避免泛泛而谈
```

#### 模板 3：来源整理型

```text
请把当前 notebook 中与[主题]相关的内容整理成：
1. 核心概念
2. 研究方法
3. 代表用途
4. 可直接用于《狼厅》人物分析的切入点
```

### NotebookLM 提问原则

- 问方法，不直接问小说细节。
- 强调“按方法分类”“按研究对象分类”，不要只要一段泛泛总结。
- 明确要求区分：
  - 来源扎根层
  - 应用推演层
- 如果要写进论文，要求它输出“研究对象 / 常用技术 / 代表价值 / 适用场景”四段结构。

### 推荐问法

- “数字人文如何研究文学人物？”
- “人物社会网络分析适合回答什么问题？”
- “大模型如何用于叙事聚焦分析？”
- “人格画像如何避免落成简单道德评判？”

### 不推荐问法

- “安妮·博林到底是不是坏人？”
- “帮我随便总结一下人物研究方法”
- “直接告诉我论文怎么写”

这些问法太散，结果通常不稳定。

## 二、GraphRAG / query_ollama.py 怎么问

### 适用场景

- 直接问索引后的《狼厅》三部曲文本
- 补人物卡
- 补事件卡
- 抽人物关系、权力结构、风格特征、叙事特征
- 生成“对象层”的分析材料

### 基本命令

工作目录：

```powershell
conda activate Graph
cd H:\graphrag-0.5.0\ragtest
```

全局问法：

```powershell
python query_ollama.py --root . --method global "安妮·博林在《狼厅》三部曲中是如何被塑造成一个政治人物的？"
```

局部问法：

```powershell
python query_ollama.py --root . --method local "安妮·博林与托马斯·克伦威尔的关系是如何发展并逐渐恶化的？"
```

### `global` 和 `local` 的分工

- `global`
  - 适合问全局判断
  - 适合问人物整体形象、人物命运、婚姻如何改变结构、某事件的整体意义
  - 适合问“为什么”“如何一步步变化”“综合分析”
- `local`
  - 适合问局部细节
  - 适合问关系恶化、语言风格、姿态、场景中的权力表现、私人记忆影响
  - 适合问“有哪些具体表现”“哪些细节最能体现”

简化规则：

- 问结构，用 `global`
- 问细节，用 `local`

### GraphRAG 提问模板

#### 模板 1：人物整体塑造

```text
[人物]在《狼厅》三部曲中是如何被塑造成一个[政治人物/复杂人物/制度节点]的？
他/她为什么不只是[常见刻板身份]，而是[更关键的结构性角色]？
```

建议：`global`

#### 模板 2：关系变化

```text
[人物A]与[人物B]的关系是如何发展并逐渐恶化/巩固的？
有哪些具体的政治冲突、个人 tensions 和策略分歧导致这一变化？
```

建议：`local`

#### 模板 3：人物风格

```text
[人物]在言谈、气质、姿态和出场方式上有哪些鲜明特征？
这些特征如何体现他/她的权力风格、野心、敏感与威胁感？
```

建议：`local`

#### 模板 4：崛起与毁灭

```text
[人物]的崛起依赖了哪些条件？
他/她的毁灭又是由哪些力量共同造成的？
请结合宫廷政治、性别期待、王朝继承压力和人物关系网络进行分析。
```

建议：`global`

#### 模板 5：重塑人物形象

```text
《狼厅》三部曲是如何重新塑造[人物]形象的？
小说在哪些方面突破了传统刻板印象，并把他/她写成一个怎样的复杂人物？
```

建议：`global`

#### 模板 6：统治者研究

```text
[统治者]被塑造成一个怎样的统治者？
他/她在权威、欲望、情绪、表演性和政治判断上有哪些核心特征？
```

建议：`global`

### GraphRAG 提问原则

- 一次只问一个问题，不要在一条命令里塞多个独立任务。
- 每次只围绕一个人物或一组强关联人物。
- 问题中直接写出分析维度，避免让模型自己猜。
- 尽量使用：
  - “如何被塑造成”
  - “为什么不只是”
  - “有哪些具体表现”
  - “请综合分析”
- 如果 `global` 返回空答案或命中很差：
  - 先不要硬信
  - 改成 `local` 重跑一次
  - 再把结果整理后回写

### 推荐工作流

1. 先写 5 个稳定问题
2. 用 `global` 跑整体问题
3. 用 `local` 跑关系、风格、场景和细节问题
4. 过滤明显不稳的细节
5. 压缩回人物卡，不直接整段复制模型原答案

## 三、以后补人物卡的标准五问

这套五问已经在安妮·博林、托马斯·克伦威尔、亨利八世身上验证过，可以复用。

### 普通核心人物

1. `[人物]在《狼厅》三部曲中是如何被塑造成一个政治人物/复杂人物的？`
2. `[人物]与[关键对手或盟友]的关系是如何发展并变化的？`
3. `[人物]在言谈、气质、姿态和出场方式上有哪些鲜明特征？`
4. `[人物]的崛起依赖了哪些条件，他/她的毁灭又由哪些力量共同造成？`
5. `《狼厅》三部曲是如何重新塑造[人物]形象的？`

### 统治者或最高权力人物

1. `[人物]被塑造成一个怎样的统治者？`
2. `[人物]与[核心执行者]的关系如何演变？`
3. `[人物]的婚姻/任命/清洗如何改变宫廷结构？`
4. `[人物]如何通过语言、礼仪、场面控制来维持权力形象？`
5. `[人物]的私人欲望如何影响国家政治？`

## 四、结果怎么回写进人物卡

不要直接复制模型原答案。建议压到这几个固定栏目：

- `基于小说文本问答的补充`
- `可直接用于论文的压缩判断`
- 必要时更新：
  - `社会网络分析`
  - `人格画像`
  - `话语与聚焦`
  - `关键判断`

回写原则：

- 保留判断，不保留废话
- 保留结构，不保留重复
- 保留与人物卡栏目匹配的句子
- 对明显不稳的细节做过滤

## 五、一个最稳的组合

以后如果要做新人物，直接按这个顺序：

1. `NotebookLM` 问方法
2. `GraphRAG global` 问整体塑造
3. `GraphRAG local` 问关系与风格
4. 回写人物卡
5. 再把可复用结论挂回方法页或案例页
