# 事件研究提问指南

## 目的

这页用于沉淀以后可重复使用的事件提问路径，尤其是已经在 `国王大事` 上验证过的文本检索命令方式。

简化理解：

- 先用 `NotebookLM` 问“事件研究应该怎样做”
- 再用 `query_ollama.py` 问“小说文本里这个事件是怎样被写出来的”

## 已验证的工作目录与环境

```powershell
conda activate Graph
cd H:\graphrag-0.5.0\ragtest
```

## 当前已验证的稳定命令

目前已验证可稳定运行的是 `global` 问法，命令形式如下：

```powershell
python query_ollama.py --root . --method global "Why is the King's Great Matter the foundational political event of the Wolf Hall trilogy rather than just Henry VIII's divorce case?"
```

这次验证说明：

- 当前目录 `H:\graphrag-0.5.0\ragtest` 下的 `--root .` 是可用的。
- 当前环境实际调用的是本地 Ollama 模型 `phi4:14b-q8_0`。
- 对事件层来说，`global` 目前比 `local` 更稳。

## 当前不稳定项

- `local` 查询目前会调用 embedding 流程。
- 当前环境里这一部分会因为 `text-embedding-3-small` 不可用而失败。
- 因而事件层现阶段应优先使用 `global`，先获取整体判断、阶段链条和叙事呈现，再回写事件卡。

## 事件提问的推荐顺序

### 1. 先问事件为何重要

```powershell
python query_ollama.py --root . --method global "Why is [event] a foundational political event in the Wolf Hall trilogy rather than just [surface description]?"
```

用途：

- 拿事件定义
- 拿结构地位
- 拿“为什么不只是……”这一层判断

### 2. 再问事件如何升级

```powershell
python query_ollama.py --root . --method global "What stages turn [event] from [starting condition] into [broader crisis] in the Wolf Hall trilogy?"
```

用途：

- 拿阶段链条
- 拿因果升级路径
- 回写 `事件链与阶段`

### 3. 再问事件如何被叙事呈现

```powershell
python query_ollama.py --root . --method global "How is [event] narratively presented in the Wolf Hall trilogy: as [option A], [option B], [option C], or [option D]?"
```

用途：

- 拿叙事层判断
- 拿“它在小说里像什么，而不只是历史上是什么”这一层
- 回写 `叙事呈现`

## 已验证的国王大事问题

### 问题 1：事件为何重要

```powershell
python query_ollama.py --root . --method global "Why is the King's Great Matter the foundational political event of the Wolf Hall trilogy rather than just Henry VIII's divorce case?"
```

### 问题 2：阶段与升级路径

```powershell
python query_ollama.py --root . --method global "What stages turn the King's Great Matter from a dynastic concern into a constitutional and religious crisis in the Wolf Hall trilogy?"
```

### 问题 3：叙事呈现

```powershell
python query_ollama.py --root . --method global "How is the King's Great Matter narratively presented in the Wolf Hall trilogy: as private desire, legal process, dynastic anxiety, or state transformation?"
```

## 回写原则

- 事件卡是给别人看的，不是技术日志。
- 保留判断，不保留模型废话。
- 保留阶段、结构、叙事方式，不原样堆砌英文输出。
- 明显过泛、重复或不稳的内容要压缩或丢弃。

## 最稳的事件工作流

1. 在 [[08 Sources/NotebookLM-数字人文事件研究方法]] 确认事件分析框架。
2. 在当前目录用 `global` 英文问题跑三次：重要性、阶段、叙事呈现。
3. 把结果压回事件卡：
   - `事件定义`
   - `事件链与阶段`
   - `叙事呈现`
   - `可直接用于论文的压缩判断`
4. 如果后续 `local` 环境修好，再补参与者结构与更细节的局部问题。
