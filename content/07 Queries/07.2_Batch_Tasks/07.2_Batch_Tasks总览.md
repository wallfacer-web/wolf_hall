# 07.2 Batch Tasks总览

> [!abstract] 核心定位
> `07.2_Batch_Tasks` 负责处理同一类对象的规模化补厚。  
> 它不是单个案例页，也不是横向矩阵页，而是把一批人物 / 制度 / 概念 / 事件放进同一套流程中统一推进。

> [!info] 使用原则
> `QBatch` 页优先解决三类问题：
> `批量问法复用`
> `统一回写格式`
> `阶段性执行追踪`

## 推荐工作流

1. 先确定这一批对象共享的核心问题。
2. 再统一生成 `NotebookLM` 与 `GraphRAG` 问法。
3. 按对象逐项执行，并在页内更新勾选状态。
4. 把结果压缩回对象页或理论页。
5. 在批处理页记录残留缺口，而不是把长答案堆在对象页里。

## 和其他查询层的分工

- `QBatch`
  - 处理“同类对象如何统一补厚”。
- `QCase`
  - 处理“单个案例如何形成完整证据链”。
- `QCross`
  - 处理“多个对象如何形成中层结构判断”。
- `QWS`
  - 处理“问题还没稳定之前的试问与实验”。
