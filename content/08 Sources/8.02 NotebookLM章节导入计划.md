# NotebookLM章节导入计划

> [!abstract] 导入计划定位
> 这页用于把书目压成 `优先问题链 -> 章节单元 -> 导入重点 -> 回写去向` 的执行计划，而不是单纯列阅读清单。

> [!info] 快速导航
> [[#第一优先级：立刻可服务当前页面|第一优先级]] · [[#第二优先级：可扩展当前主线|第二优先级]] · [[#第三优先级：方法与补充来源|第三优先级]] · [[#执行原则|执行原则]]

| 阅读轴 | 关注点 |
| --- | --- |
| 计划 | 当前先导哪些书、哪些章节 |
| 问题链 | 每轮导入准备服务哪条主线 |
| 回写 | 导入后具体写回哪些页面 |
| 约束 | 如何避免无边界堆料 |

这张计划页的目标不是一次性把整本书塞进 `NotebookLM`，而是把扩展书单拆成可问题驱动、可批量执行、可回写的导入单元。

原则只有三条：

- 优先导入与当前 `QCase / QBatch / QCross` 直接相关的章节。
- 每次导入只服务一条明确问题链，不做无边界堆料。
- 导入后必须回写到具体页，不允许只停留在 `NotebookLM` 输出层。

## 第一优先级：立刻可服务当前页面

### A. 用于 `QCase_国王大事与英格兰脱离罗马`

#### 1. J. J. Scarisbrick, *Henry VIII*

- 导入重点：
  - `King's Great Matter`
  - Henry VIII 与 Catherine of Aragon
  - papal jurisdiction / annulment / royal supremacy
- 主要用途：
  - 补 `国王大事` 的事件链和法理背景
  - 区分“离婚问题”“合法性问题”“主权重组问题”三层
- 回写目标：
  - [[7.3.03 QCase_国王大事与英格兰脱离罗马]]
  - [[3.02 国王大事|国王大事]]
  - [[4.06 英格兰教会|英格兰教会]]

#### 2. G. R. Elton, *The Tudor Revolution in Government*

- 导入重点：
  - Cromwell 与行政重组
  - conciliar / parliamentary / bureaucratic change
  - statute、record、paperwork 与国家执行链
- 主要用途：
  - 把 `国王大事` 从婚姻纠纷推进成国家机器重组
  - 补强你现在库里反复出现的“纸与墨”“记录”“文书权力”
- 回写目标：
  - [[7.3.03 QCase_国王大事与英格兰脱离罗马]]
  - [[4.05 议会立法|议会立法]]
  - [[4.01 财政与行政官僚系统|财政与行政官僚系统]]

#### 3. Diarmaid MacCulloch, *Thomas Cromwell*

- 导入重点：
  - Cromwell 在国王大事中的角色
  - 与 Wolsey、Henry、Gardiner、Anne Boleyn 的位置变化
  - 行政与宗教改革接口
- 主要用途：
  - 给 `国王大事` 页补克伦威尔的行动位置，而不是只写宏观事件
- 回写目标：
  - [[2.03 托马斯·克伦威尔|托马斯·克伦威尔]]
  - [[7.3.03 QCase_国王大事与英格兰脱离罗马]]

## 建议导入顺序

如果按当前最紧要的任务来排，建议这样跑：

1. `Henry VIII`
2. `The Tudor Revolution in Government`
3. `Thomas Cromwell`
4. `Wolf Hall`
5. `Bring Up the Bodies`
6. `The Mirror & the Light`
7. `Narratology`
8. `The Digital Humanities Coursebook`

## 导入后必须补的记录字段

每次导入建议至少记录：

- 书名
- 作者
- 导入章节或页码范围
- 本次服务的问题
- 得到的 3 到 5 条有效判断
- 回写到了哪些页面
- 是否还需要二次追问

## 后续动作

- 有书后先建立每本书的 `章节摘录清单`
- 每本书第一次只导入最相关的 1 到 3 章
- 导入结果优先回写 `QCase_国王大事与英格兰脱离罗马`
