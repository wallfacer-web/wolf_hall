# NotebookLM-数字人文事件研究方法

> [!abstract] 页面定位
> 这页用于约束 `NotebookLM` 怎样服务事件研究。  
> 它的重点不是“补充史实越多越好”，而是把事件压成 `阶段链 + 机制链 + 回写路径`。

> [!info] 适用对象
> 主要服务于 [[1.09 事件分析方法|事件分析方法]]、[[3.00 事件总览|事件总览]] 与各张事件页，尤其适合国家转型、政治清洗和制度重组类事件。

| 阅读轴 | 关注点 |
| --- | --- |
| 事件 | 这类页最该压出的结构是什么 |
| 提问 | 向 NotebookLM 稳定追问哪些层次 |
| 机制 | 如何从摘要转成机制链与阶段链 |
| 去向 | 最终回写到哪些事件页和案例页 |

## 一、这页解决什么问题

- 防止事件页只停留在情节复述
- 让 `NotebookLM` 输出更适合回写到 `03 Events`
- 为 `QCase`、`QCross` 和制度页准备可压缩的结构判断

## 二、事件研究的五个核心维度

### 1. 事件切分

- 这个事件能否拆成若干阶段
- 哪些节点才是真正的起点、转折和终局

### 2. 因果机制

- 事件如何被长期酝酿出来
- 哪些导火索把潜在冲突变成正式行动

### 3. 参与者结构

- 谁是推动者、执行者、替代者、被波及者
- 参与者是如何被组织进事件链的

### 4. 时间与空间

- 事件在哪些空间发生推进
- 时间是突进、拖延、回返还是分阶段收束

### 5. 叙事呈现

- 小说把它写成危机、管理过程、审判程序，还是情感崩塌
- 读者看到的是结果，还是看到国家机器如何运作

## 三、向 NotebookLM 提问时的稳定问法

### A. 阶段链问法

- 请把这个事件拆成 5 到 7 个阶段，每个阶段给出一句功能说明。
- 如果把这件事写成“从酝酿到收束”的过程，中间哪些阶段不能省略？

### B. 机制链问法

- 这个事件最核心的机制链是什么？
- 它更像 `gossip -> interrogation -> confession -> indictment`，还是 `delay -> negotiation -> pardon illusion -> purge` 这类结构？

### C. 参与者问法

- 谁真正推动了事件，谁只是名义上的中心？
- 哪些人物在这件事中承担工具性角色？

### D. 空间与制度问法

- 哪些地点不只是背景，而是不同权力程序的空间节点？
- 这个事件涉及哪些制度装置，例如誓言、法案、供词、修道院财产、议会程序？

## 四、推荐输出格式

每次试问尽量压成五栏：

- `阶段链`
- `机制链`
- `关键参与者`
- `空间 / 制度节点`
- `可回写页面`

## 六、事件研究的常见失误

- 把事件写成时间顺序摘要，而不是结构分析
- 只说结果，不写机制链
- 只谈核心人物，不谈制度工具
- 只堆史实，不压缩成可回写判断

## 七、最小工作流

1. 先确定目标事件与研究问题
2. 用 `NotebookLM` 拿到阶段链和机制链
3. 再补参与者、空间和制度节点
4. 把输出压进事件页与 `QCase`
5. 最后回看是否可联动制度页或概念页
